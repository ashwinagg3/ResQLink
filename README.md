# ResQLink - Premium Emergency Contact System

**ResQLink** is a high-fidelity, full-stack safety application designed to provide instant assistance during emergencies. Built with a focus on tactical efficiency and a premium visual experience, it empowers users to manage emergency contacts, trigger SOS alerts, and access official emergency channels seamlessly.

![ResQLink Banner](https://img.shields.io/badge/Status-Complete-brightgreen)
![Tech Stack](https://img.shields.io/badge/Stack-Node.js%20|%20Express%20|%20MongoDB%20|%20VanillaJS-blue)

## ✨ Premium UI/UX Experience
ResQLink features a sophisticated **Tactical Light Aesthetic** designed for clarity and impact:
- **3D Background Elements:** Dynamic floating primitives and mesh gradients for visual depth.
- **Micro-Animations:** Smooth transitions, pulsing SOS triggers, and hover-active tactical panes.
- **Glassmorphism:** Premium frosted-glass headers and cards for a modern, high-end feel.
- **Responsive Layouts:** Optimized for both desktop and mobile emergency usage.

## 🚀 Core Features

### 🔐 User Ecosystem
- **Instant Authentication:** Seamless register/login workflow.
- **Tactical Profile Management:** Users can maintain vital information including **Blood Group**, **Phone Number**, and **Medical Conditions** for first responders.

### 🆘 SOS System
- **High-Impact Trigger:** A prominent, pulsing SOS button on the dashboard for immediate action.
- **Confirmation Safety:** Double-tap confirmation modal to prevent accidental triggers.
- **Intelligent Logging:** Every alert is logged with a timestamp and a record of which contacts were notified.

### 📞 Contact & Resource Management
- **Contact Manager:** Full CRUD (Create, Read, Update, Delete) capability for managing personal emergency contacts.
- **Official Emergency Directory:** A curated list of official Indian emergency numbers (112, 100, 101, 102, 1091, etc.) with instant-call functionality.
- **Alert History:** Sequential timeline of all past SOS incidents for review and reporting.

## 🛠 Technical Architecture

### Backend (Node.js & Express)
- **Modular Routing:** Separate routes for Auth, Alerts, and Contacts for scalability.
- **MongoDB Integration:** Mongoose ODM for reliable data schema management.
- **Static Asset Serving:** Built-in serving of the high-fidelity frontend directly from the server.

### Frontend (Vanilla JS & Modern CSS)
- **Google Fonts:** Utilizing 'Inter' for maximum readability.
- **Material Symbols:** Professional-grade iconography from Google Material Design.
- **Utility CSS:** Leveraging Tailwind for layout precision without compromising the custom vanilla aesthetic.

## 📥 Setup & Installation

### 1. Prerequisites
- [Node.js](https://nodejs.org/) (v14+ recommended)
- [MongoDB](https://www.mongodb.com/try/download/community) installed and running locally.

### 2. Installation
Clone the repository and install dependencies for the backend:
```bash
cd backend
npm install
```

### 3. Database Configuration
By default, the application connects to:
`mongodb://127.0.0.1:27017/emergency_contact_system`
Ensure your local MongoDB service is active before starting the server.

### 4. Running the Application
Start the server from the `backend` directory:
```bash
node server.js
```
The application will be accessible at: **[http://localhost:5000](http://localhost:5000)**
